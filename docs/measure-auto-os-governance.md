# 統治金型「MEASURE 自動OS」

第1条（入口）: 入力権限を絞り、MEASURE OSを通さないモノの移動を禁止。

第2条（キャパ）: 1日の上限超過時に画面警告＋管理者通知。

第3条（締切）: 事務、営業の指定時刻（例: 15時）以降の入力は、日付を「翌々日（翌々営業日）」へ自動置換する。  
本ルールにより、当日および翌日の事務、営業からの作業指示はシステム上登録できない。  
締切以降に発生した緊急案件は第6条（緊急）として処理し、通常指示としては扱わない。

第4条（監査）: パレートの法則で2割の重要品目は定期、残り8割は指定数を毎日ランダム抽出して棚卸し指示。ズレ検知で「未修正フラグ」を立て、修正されるまで維持。※猶予判定は2-①に従う。  
【修正権限の分離】実数の最終修正および「未修正フラグ」の解除は、現場ではなく事務（管理者）のみが実行できる権限設計とすること。事務が原因を特定し修正を完了するまで、異常値としてダッシュボードに維持され続ける仕組みを構築すること。

第5条（予告と着手と実績）:

予告（現場主導型）: 現場担当者が、第7条の優先度ソートを参考に「明日これを行う」という段取りを自ら入力（掲示）。前日残や保留解除分も含め、現場の判断による作業計画をシステム上に登録する。  
着手: 前の作業による不備や破損がないかを確認し、異常がなければ押すこと。ボタンを押した瞬間から、その後の現場責任はすべて本人（着手者）に移ります。  
実績: 予告に対し「実績数」を入力。  
通知: 数値乖離、見通し違い、予告なし実績、および「第4条の修正放置品の予告入力」を検知。※異常発生後の赤色昇格は2-①に従う。

第6条（緊急）: 在庫不足・故障等は全員へ即時通知。

第7条（製造特化）: 業務課入力の「納期・目的」を納期順にソートし、現場用ビューで優先度を自動色分け（赤: 納期直前、黄: 在庫払底、青: 通常）
